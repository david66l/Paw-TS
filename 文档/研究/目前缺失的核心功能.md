 缺失的（影响日常使用）
  
  优先级: P1
  功能: Ollama 本地模型
  说明: settings.local.json 有 ollama_host 字段，但 models 包没有 OllamaCompatibleModel 实现，无法跑本地模型
  ────────────────────────────────────────
  优先级: P1
  功能: 并行工具调用
  说明: Orchestrator 一次只执行一个工具。Claude Code 支持同时 read 多个文件、grep + read 并行，能显著减少等待
  ────────────────────────────────────────
  优先级: P1
  功能: Batch edit / apply_patch
  说明: 没有多文件同时编辑的工具。AI 改 10 个文件需要 10 次 tool call，每次都要等模型响应
  ────────────────────────────────────────
  优先级: P1
  功能: Streaming shell 输出
  说明: run_shell 用 spawnSync（阻塞到结束），无法实时看到长时间命令的输出（如 npm install）
  ────────────────────────────────────────
  优先级: P2
  功能: Checkpoint / rollback
  说明: 没有代码快照和回滚。Claude Code 会在每次 tool 调用前自动保存状态，出错可以 /undo
  ────────────────────────────────────────
  优先级: P2
  功能: Worktree 隔离
  说明: 没有 git worktree 支持。无法在一个干净的分支/工作树上运行 agent，避免污染当前工作区
  ────────────────────────────────────────
  优先级: P2
  功能: Session replay UI
  说明: TUI 没有查看历史会话列表和重放的功能。有存储（session-store），但没有交互界面
  ────────────────────────────────────────
  优先级: P2
  功能: 设置向导
  说明: 没有 paw-ts init 或交互式配置流程。用户需要手动创建 .paw/settings.local.json
  ────────────────────────────────────────
  优先级: P3
  功能: 代码索引 / 语义搜索
  说明: 只有基于 grep 的文本搜索，没有 AST 解析或向量索引。大型代码库中找相关文件效率低
  ────────────────────────────────────────
  优先级: P3
  功能: 文件系统监控
  说明: 没有 fs.watch 集成。外部编辑器修改文件后，agent 不知道内容已变
  ────────────────────────────────────────
  优先级: P3
  功能: Web App
  说明: apps/web/ 几乎是空的，没有实际功能
  ────────────────────────────────────────
  优先级: P3
  功能: IDE 插件
  说明: 没有 VS Code / JetBrains 扩展
  ────────────────────────────────────────
  优先级: P3
  功能: 图片/二进制处理
  说明: read_file 只读文本，无法处理图片、PDF 等
  ────────────────────────────────────────
  优先级: P3
  功能: 成本追踪按模型定价
  说明: CostTracker 目前用所有模型的平均价格估算，没有按实际使用的模型匹配定价
