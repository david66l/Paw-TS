# 第1轮升级：工具调用协议 — 原生 function calling 优先

> 2026-06-07 | 预计 4 小时

## 目标

模型调用时优先使用原生 `tool_use`（API 结构化字段），不再依赖从文本里扫描 JSON。只有不支持原生 tool calling 的 provider 才走 fallback 文本扫描。

## 当前状态

```
invokeModelOnce (streaming):
  tool_use chunk → 转成 JSON 文本行 → 拼进 acc 字符串
  ↓
invokeModelOnce (non-streaming):
  result.text → 直接返回
  ↓
normalizeToolCalls → 映射 sanitized name → 纯文本
  ↓
parseAgentActionsFromModelText → 扫文本里所有 {} → 找 tool_call
```

问题：即使模型已经通过 API 结构化字段告诉你了 tool name 和 args，你仍然把它转回文本、再从文本里扫 JSON。多了一步无意义的序列化-反序列化，且重新引入了 JSON 解析的所有风险（误匹配代码块、格式错误等）。

## 改动文件

| 文件 | 改动 |
|------|------|
| `packages/models/src/types.ts` | `ModelCompletionResult` 加 `toolCalls` 字段 |

## 改动细节

### 1. types.ts — 加 toolCalls 字段

```typescript
export interface ModelCompletionResult {
  readonly text: string;
  readonly usage?: ModelTokenUsage;
  readonly thinking?: string;
  readonly finishReason?: string;
  /** Native structured tool calls (when provider supports function calling). */
  readonly toolCalls?: readonly NativeToolCall[];
}

export interface NativeToolCall {
  readonly id: string;
  readonly name: string;
  readonly arguments: Record<string, unknown>;
}
```

### 2. invokeModelOnce — 收集原生 tool_use

**Streaming 路径**：当收到 `tool_use` chunk 时不转成文本行，而是收集到 `nativeToolCalls[]` 数组。`done` 时返回。

**Non-streaming 路径**：检查 `result.toolCalls`，有则直接返回。

**关键**：当有原生 tool_use 时，`text` 字段仍然保留（包含模型的推理文字），但 tool call JSON 不再混在 text 里。

### 3. executeTurn — 优先原生，fallback 扫描

```typescript
// 5. Parse actions
const knownTools = new Set(toolNameMap.values());

if (nativeToolCalls && nativeToolCalls.length > 0) {
  // 原生 tool_use：直接映射为 AgentToolCallAction
  toolCalls = nativeToolCalls.map(tc => {
    const originalName = toolNameMap.get(tc.name) ?? tc.name;
    return { type: "tool_call", tool: originalName, args: tc.arguments };
  });
  reasoningText = text; // text 里只有推理文字，没有 JSON
} else {
  // Fallback：文本扫描
  const parsed = parseAgentActionsFromModelText(text, { knownTools });
  toolCalls = parsed.actions;
  reasoningText = parsed.text;
}
```

### 4. normalizeToolCalls 保留

仍然需要——某些 provider 返回的 tool name 是 sanitized（点变下划线），需要映射回来。

## 不涉及

- `parse-agent-action.ts` 不动——它仍然是 fallback 路径，给不支持原生 tool calling 的 provider 用
- `parse-tool-call.ts` 不动
- `registry.ts:toolDefinitions()` 不动——已经生成正确的 OpenAI-format schema
